# Ploigi
FastAPIとMQTTを用いた、水耕栽培・環境制御システム。

## プロジェクト背景,経緯
2023年にハウス農家で作業に従事した際、「現場の環境制御をソフトウェアやIoTなど活用できることが非常に多いのでは」と感じ、またエンジニアとして自身の興味の引っ掛かかりどころであり、進む方向性なのではと感じたのが原点です。

### 1.現場の課題と最初の挫折
現場に縛られる作業から解放されれば楽になることが多いと感じたのでまずは「世界中のどこからでも遠隔制御できるシステム」という目標をぼんやりと決めスタートしました。
以前本を見ながら作成したWebアプリがあるという理由だけで、当初はDjango を用いてWeb アプリと自作した給水ポンプデバイスに挑戦しました。自作デバイスとブラウザからネットを介して遠隔での操作を行うという目標は達成しました。しかし、当時はインフラ知識や通信プロトコルはもちろんデバイスの知識や理解は雰囲気程度の理解しかなく、かろうじて同ネットワーク内での給水ポンプの遠隔操作はできましたが様々な問題がありました。動くものの拡張性もなくまた何かを変更すると動かないがなぜ動かないのかわからないという状態でした。

### 2. 失敗分析と再構築
それまでの試行をもとに「なぜ動かないのか」「何が必要か」「何がどのように関係しているのか」を考えながら調べ、その中で少しずつ何がわからないかの理解しようと進めていきました。サーバー制約やプロトコル選択の誤りなど多くの課題が明確になりました。この経験をもとに、現在のそれなりに適しているのではというシステム構成のアイデアへと辿り着きました。

### 3. Project "Ploigi"
遠隔地からの確実な環境制御と、柔軟な機能拡張に対応できるシステムの構築を目指します。悲しいかなやりたいことは想像程度にあるけど、世に出ているデバイスをいきなり使う知識も、そもそも資金もないので「ないものは自作」という方針で試行錯誤しながら最適な形を模索しています。自作デバイス（ESP8266等）からスタートし、最終的には世の中にある様々な汎用センサーや機器をプラグアンドプレイで組み込めるような、自由度の高いプラットフォームを目指しています。

## 構成
- **Backend**: FastAPI
- **Messaging**: MQTT (Mosquitto)
- **Edge Devices**: ESP8266 / NodeMCU
- **Infrastructure**: Raspberry Pi 4B
